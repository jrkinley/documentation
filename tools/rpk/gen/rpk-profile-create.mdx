Create an rpk profile

## Usage

```bash
rpk profile create [NAME] [flags]
```

## Flags

<table>
  <tbody>
    <tr>
      <td>
        <strong>Value</strong>
      </td>
      <td>
        <strong>Type</strong>
      </td>
      <td>
        <strong>Description</strong>
      </td>
    </tr>
    <tr>
      <td>-d, --description</td>
      <td>string</td>
      <td>Optional description of the profile.</td>
    </tr>
    <tr>
      <td>--from-cloud</td>
      <td>string</td>
      <td>
        Create and switch to a new profile generated from a Redpanda Cloud
        cluster ID.
      </td>
    </tr>
    <tr>
      <td>--from-simple</td>
      <td>string</td>
      <td>
        Create and switch to a new profile from a (simpler to define)
        `redpanda.yaml` file.
      </td>
    </tr>
    <tr>
      <td>-h, --help</td>
      <td>-</td>
      <td>Help for create.</td>
    </tr>
    <tr>
      <td>-k, --kv</td>
      <td>strings</td>
      <td>
        Create and switch to a new profile, filling profile fields with
        key=value pairs.
      </td>
    </tr>
    <tr>
      <td>--config</td>
      <td>string</td>
      <td>
        Redpanda or rpk config file; default search paths are
        ~/.config/rpk/rpk.yaml, $PWD, and /etc/redpanda/`redpanda.yaml`.
      </td>
    </tr>
    <tr>
      <td>-X, --config-opt</td>
      <td>stringArray</td>
      <td>
        Override rpk configuration settings; '-X help' for detail or '-X list'
        for terser detail.
      </td>
    </tr>
    <tr>
      <td>-v, --verbose</td>
      <td>string</td>
      <td>
        [="info"] Log level (none, error, warn, info, debug) (default "none").
      </td>
    </tr>
  </tbody>
</table>
